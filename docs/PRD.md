# PRD — 로또 스크래치 추천 앱

> 프로토타입 기준. 통계 기반 번호풀에서 스크래치 카드로 번호를 직접 선택하는 인터랙티브 로또 앱.

---

## 1. 제품 개요

### 한 줄 정의
통계 기반으로 번호풀을 좁혀주고, 사용자가 스크래치 카드로 직접 6개를 선택하는 로또 번호 선택 앱.

### 핵심 가치
- **재미**: 긁는 행위 자체가 번호 선택. 애니메이션으로 두근거림 제공
- **근거 있는 선택**: 통계 기반 번호풀(HOT/COLD/MIX)로 선택에 의미 부여
- **기록**: 내가 선택한 번호 이력 보존

---

## 2. 기능 그룹

| 그룹 | 설명 | 저장 |
|------|------|------|
| **번호 뽑기** | 모드 선택 → 스크래치 카드 → 6개 선택 | localStorage |
| **통계** | 번호별 출현 빈도, 역대 회차 조회 | Supabase (읽기) |
| **내 기록** | 내가 뽑은 번호 이력 | localStorage |

---

## 3. 번호 뽑기 (핵심 기능)

### 3-1. 모드 선택 화면

진입 시 4가지 모드 중 하나를 선택.

| 모드 | 번호풀 | 기준 |
|------|--------|------|
| `HOT` | 최근 100회차 출현 빈도 상위 20개 | 자주 나온 번호 |
| `COLD` | 최근 100회차 출현 빈도 하위 20개 | 오래 안 나온 번호 |
| `MIX` | HOT 상위 10개 + COLD 하위 10개 = 20개 | 균형 |

### 3-2. 스크래치 카드 화면

모드 선택 후 스크래치 카드 화면으로 전환.

**레이아웃**
- 번호풀에 해당하는 번호들이 그리드로 배치 (모두 가려진 상태)
- 상단: 선택된 번호 6개 슬롯 표시 (빈 슬롯 → 채워지는 방식)
- 하단: 현재 몇 개 선택했는지 표시 (`3 / 6`)

**인터랙션**
- 터치/마우스 드래그로 긁으면 번호 공개
- 공개된 번호 탭하면 선택 (슬롯에 추가)
- 선택 취소 가능 (슬롯에서 제거 → 다시 선택 가능)
- 6개 채워지면 완료 버튼 활성화

**애니메이션 요구사항**
- 긁히는 효과: 스크래치 텍스처가 벗겨지는 모션
- 번호 공개 시: 숫자가 튀어나오는 효과
- 6개 완료 시: 전체 화면 축하 이펙트 (폭죽, 반짝임)
- 선택된 번호: 골드 컬러 강조

### 3-3. 완료 화면

6개 선택 완료 후 결과 요약.

- 선택한 번호 6개 크게 표시
- 모드 정보 표시 (HOT 기반, MIX 기반 등)
- `저장하기` → 내 기록에 추가 (localStorage)
- `다시 뽑기` → 모드 선택 화면으로
- `공유하기` → 이미지로 저장 또는 클립보드 복사

---

## 4. 통계

### 4-1. 번호별 빈도 차트
- 1~45 전체 번호 출현 횟수 바 차트
- 기준 회차 필터: 최근 50 / 100 / 전체
- HOT (상위 6개) 골드, COLD (하위 6개) 블루 컬러 강조

### 4-2. 회차 조회
- 전체 회차 목록 (최신순, 페이지네이션)
- 회차 번호 / 날짜 / 당첨 번호 6개 + 보너스 / 1등 당첨금 표시

---

## 5. 내 기록

- 저장한 번호 이력 목록 (최신순)
- 저장 날짜 / 선택 모드 / 번호 6개 표시
- 개별 삭제 가능
- localStorage 저장 (앱 삭제 또는 브라우저 데이터 초기화 시 소멸)

---

## 6. 사용자 플로우

```
앱 진입
  ├── 번호 뽑기 탭
  │     → 모드 선택 (HOT / COLD / MIX / ALL)
  │     → 스크래치 카드 (긁기 → 번호 선택 × 6)
  │     → 완료 화면 (저장 / 다시뽑기 / 공유)
  │
  ├── 통계 탭
  │     → 빈도 차트 (필터 선택)
  │     → 회차 조회 목록
  │
  └── 내 기록 탭
        → 저장된 번호 목록
        → 삭제
```

## 7. BDD 시나리오

```gherkin
Feature: 번호 뽑기

  Scenario: HOT 모드로 스크래치 카드 진입
    Given 최근 100회차 데이터가 DB에 있을 때
    When  사용자가 HOT 모드를 선택하면
    Then  출현 빈도 상위 20개 번호가 스크래치 카드에 배치된다
    And   모든 번호는 가려진 상태로 시작한다

  Scenario: 번호 6개 선택 완료
    Given 스크래치 카드 화면에 있을 때
    When  사용자가 6개 번호를 긁고 선택하면
    Then  완료 애니메이션이 실행된다
    And   완료 화면으로 전환된다

  Scenario: 내 기록 저장
    Given 완료 화면에 있을 때
    When  사용자가 저장하기를 누르면
    Then  선택한 번호 6개와 모드, 날짜가 localStorage에 저장된다
```
